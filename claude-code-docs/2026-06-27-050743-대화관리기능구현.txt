  질문 12: 스레드 삭제 시 하위 Chat 처리

  스레드를 삭제할 때 해당 스레드의 Chat들을 어떻게 처리할 건가요?

  - A) Hard delete cascade: 스레드 삭제 시 DB 레벨 ON DELETE CASCADE로 Chat도
  함께 삭제. 단순하지만 복구 불가.
  - B) Soft delete: deletedAt 컬럼 추가, 실제 삭제 없이 플래그만. 복구
  가능하지만 모든 쿼리에 WHERE deletedAt IS NULL 추가 필요.
  - C) Hard delete, 애플리케이션 레벨 cascade: DB cascade 없이 서비스에서 Chat
  먼저 삭제 후 Thread 삭제.

  내 추천: A안. 데모 목적에서 복구 기능은 불필요하고, DB cascade가 가장 안전하고
  단순합니다. 피드백 테이블도 ON DELETE CASCADE 연결하면 orphan 데이터 걱정
  없음.

  어떻게 하실 건가요?

  질문 12-1 (연계): Soft Delete 범위

  Soft delete를 선택하셨습니다. 범위를 명확히 해야 합니다.

  - A) Thread만 soft delete, Chat은 그대로 유지: Thread deletedAt 플래그만 추가.
  Chat은 삭제 플래그 없음 — Thread가 숨겨지면 자동으로 접근 불가.
  - B) Thread + Chat 모두 soft delete: Thread, Chat 각각 deletedAt 추가. 독립적
  Chat 숨김 가능하지만 복잡도 증가.

  그리고 소프트 삭제된 Thread에 속한 Chat의 피드백은 어떻게 처리할 건가요?
  - i) 피드백 조회 시 삭제된 Thread의 Chat도 포함: 피드백은 독립적으로 존재
  - ii) 삭제된 Thread의 Chat 피드백은 조회 제외: Thread 삭제 시 사실상 피드백도
  숨김

  내 추천: A안 + ii안. Thread만 soft delete하면 쿼리 복잡도를 최소화하면서 복구
  가능성을 확보할 수 있습니다. 피드백 조회 시 JOIN thread WHERE thread.deletedAt
  IS NULL 조건 추가.

  어떻게 하실 건가요?

  - A) 단일 엔드포인트, 컨트롤러에서 Any 반환: if (isStreaming) return 
  SseEmitter() else return ResponseEntity(...). Spring이 타입에 따라 처리.
  코드가 지저분해짐.
  - B) 단일 엔드포인트, 서비스 레이어에서 분기: 컨트롤러는 SseEmitter만 반환,
  비스트리밍도 SseEmitter에 한 번에 데이터 전송 후 complete. 응답 타입 통일.
  - C) 엔드포인트 분리: POST /chats (비스트리밍), POST /chats/stream (스트리밍).
  명세에는 같은 엔드포인트지만 구현이 깔끔해짐.

  내 추천: B안. SseEmitter로 통일하면 컨트롤러 반환 타입이 일관되고,
  비스트리밍은 응답을 한 번에 emit 후 complete() 호출로 처리합니다. 클라이언트
  입장에서 동일한 엔드포인트로 SSE를 처리하면 됩니다.

  질문 14: 스레드 삭제 시 캐시 무효화

  ConcurrentHashMap<UserId, Instant>에 유저의 마지막 Chat 시각이 캐시됩니다.
  그런데 해당 스레드가 소프트 삭제되면 캐시가 오염됩니다.

  예시:
  1. 유저가 오후 2:00에 마지막 Chat → 캐시에 2:00 저장
  2. 유저가 해당 Thread를 삭제 (soft delete)
  3. 유저가 오후 2:20에 새 질문 → 캐시에서 2:00 읽음 → "30분 이내"로 판단 →
  삭제된 Thread에 Chat 추가하려 시도 → 버그

  이를 어떻게 처리할 건가요?

  - A) 스레드 삭제 시 캐시에서 해당 유저 엔트리 제거: cache.remove(userId) 호출.
  다음 요청 시 DB에서 재조회 (삭제된 Thread 제외 후 MAX(createdAt))
  - B) 캐시 무시, 스레드 존재 여부를 항상 DB에서 검증: 캐시 읽은 후 해당
  Thread가 삭제됐는지 추가 확인. 캐시 효과 반감.

  내 추천: A안. 스레드 삭제 시 cache.remove(userId) 한 줄이면 해결됩니다. 30분
  캐시 목적이 DB 조회 회피인데, 삭제는 드문 이벤트라 캐시 invalidation 비용이
  무시할 수준입니다.

     Context

     빈 프로젝트(빌드 파일·소스 없음)에 Kotlin/Spring Boot 기반 AI 챗봇 서비스의
     핵심 기능인 대화(Chat) 관리 기능을 처음부터 구현한다. Grilling 세션에서
     확정된 18개 설계 결정을 모두 반영한다.

     핵심 결정 요약:
     - AI: 표준 Gemini API (API Key), gemini-3.5-flash / 임베딩
     gemini-embedding-2 (3072차원)
     - SDK: com.google.ai.client.generativeai (Google 공식 Kotlin SDK)
     - 스트리밍: 단일 엔드포인트 SseEmitter, 항상 SSE 반환 (비스트리밍도 1회

     DataSeeder로 시드
     - 페이지네이션: Thread 단위 (N threads/page)

     - model 파라미터: application.yml 화이트리스트, 기본값 gemini-3.5-flash

     2. DB 마이그레이션 (V1__init.sql)

     CREATE EXTENSION IF NOT EXISTS vector;

     CREATE TABLE users (
         id          BIGSERIAL PRIMARY KEY,
         email       VARCHAR(255) NOT NULL UNIQUE,
         password    VARCHAR(255) NOT NULL,
         name        VARCHAR(255) NOT NULL,
         role        VARCHAR(50)  NOT NULL DEFAULT 'MEMBER',
         created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
     );

     CREATE TABLE threads (
         id          BIGSERIAL PRIMARY KEY,
         user_id     BIGINT       NOT NULL REFERENCES users(id),
         created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
         deleted_at  TIMESTAMPTZ
     );

     CREATE TABLE chats (
         id          BIGSERIAL PRIMARY KEY,
         thread_id   BIGINT       NOT NULL REFERENCES threads(id),
         question    TEXT         NOT NULL,
         answer      TEXT         NOT NULL,
         created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
     );

     CREATE TABLE feedbacks (
         id          BIGSERIAL PRIMARY KEY,
         user_id     BIGINT       NOT NULL REFERENCES users(id),
         chat_id     BIGINT       NOT NULL REFERENCES chats(id),
         is_positive BOOLEAN      NOT NULL,
         status      VARCHAR(50)  NOT NULL DEFAULT 'PENDING',
         created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
         UNIQUE(user_id, chat_id)
     );

     CREATE TABLE document_chunks (
         id          BIGSERIAL PRIMARY KEY,
         content     TEXT         NOT NULL,
         embedding   vector(3072),
         source      VARCHAR(255),
         chunk_index INT
     );

     7. ThreadCacheService

     @Service
     class ThreadCacheService(
         private val threadRepository: ThreadRepository,
         private val chatRepository: ChatRepository
     ) {
         private val cache = ConcurrentHashMap<Long, Instant>()

         fun getOrCreateThread(userId: Long): ChatThread {
             val lastChatTime = cache[userId]
                 ?: chatRepository.findLatestChatTimeByUserId(userId)?.also {
     cache[userId] = it }

             val createNew = lastChatTime == null ||
                 lastChatTime.plusSeconds(1800).isBefore(Instant.now())

             return if (createNew) {
                 threadRepository.save(ChatThread(userId = userId))
             } else {
                 threadRepository.findTopByUserIdAndDeletedAtIsNullOrderByCreate
     dAtDesc(userId)
                     ?: threadRepository.save(ChatThread(userId = userId))
             }
         }

         fun updateCache(userId: Long, time: Instant) { cache[userId] = time }
         fun invalidate(userId: Long) { cache.remove(userId) }
     }

     8. RagService

     document_chunks 테이블은 JPA 엔티티 없이 JdbcTemplate으로만 접근한다
     (vector 타입 호환 이슈 방지).

     @Service
     class RagService(
         private val props: GeminiProperties,
         private val jdbcTemplate: JdbcTemplate
     ) {
         private val embeddingModel by lazy {
             GenerativeModel(modelName = props.embeddingModel, apiKey =
     props.apiKey)
         }

         suspend fun embedText(text: String): FloatArray {
             val response = embeddingModel.embedContent(text)
             return response.embedding.values
         }

         fun searchSimilarChunks(embedding: FloatArray): List<String> {
             val vectorStr = "[${embedding.joinToString(",")}]"
             return jdbcTemplate.queryForList(
                 "SELECT content FROM document_chunks ORDER BY embedding <=> 
     CAST(? AS vector) LIMIT 3",
                 String::class.java, vectorStr
             )
         }
     }

     9. ChatService

     @Service
     class ChatService(
         private val props: GeminiProperties,
         private val threadCacheService: ThreadCacheService,
         private val chatRepository: ChatRepository,
         private val threadRepository: ThreadRepository,
         private val ragService: RagService
     ) {
         // 대화 생성
         fun createChat(userId: Long, req: ChatCreateRequest, emitter: 
     SseEmitter) {
             val modelName = req.model?.also {
                 require(it in props.allowedModels) { "허용되지 않은 모델: $it"
     }
             } ?: props.chatModel

             val thread = threadCacheService.getOrCreateThread(userId)
             val history =
     chatRepository.findByThreadIdOrderByCreatedAtAsc(thread.id)

             CoroutineScope(Dispatchers.IO).launch {
                 try {
                     val embedding = ragService.embedText(req.question)
                     val chunks = ragService.searchSimilarChunks(embedding)
                     val systemInstruction = buildSystemInstruction(chunks)

                     val model = GenerativeModel(
                         modelName = modelName,
                         apiKey = props.apiKey,
                         systemInstruction = content { text(systemInstruction) }
                     )
                     val contents = buildContents(history, req.question)
                     val sb = StringBuilder()

                     if (req.isStreaming) {

     model.generateContentStream(*contents.toTypedArray()).collect { resp ->
                             val chunk = resp.text ?: ""
                             sb.append(chunk)
                             emitter.send(SseEmitter.event().data(chunk))
                         }
                     } else {
                         val resp =
     model.generateContent(*contents.toTypedArray())
                         val text = resp.text ?: ""
                         sb.append(text)
                         emitter.send(SseEmitter.event().data(text))
                     }

                     val chat = chatRepository.save(
                         Chat(threadId = thread.id, question = req.question,
     answer = sb.toString())
                     )
                     threadCacheService.updateCache(userId, chat.createdAt)
                     emitter.complete()
                 } catch (e: Exception) {
                     emitter.completeWithError(e)
                 }
             }
         }

         // 대화 목록 조회 (Thread 단위 페이지네이션)
         fun listChats(userId: Long, isAdmin: Boolean, pageable: Pageable):
     Page<ThreadWithChatsResponse> {
             val threads = if (isAdmin)
                 threadRepository.findAllByDeletedAtIsNull(pageable)
             else
                 threadRepository.findAllByUserIdAndDeletedAtIsNull(userId,
     pageable)

             return threads.map { thread ->
                 val chats =
     chatRepository.findByThreadIdOrderByCreatedAtAsc(thread.id)
                 ThreadWithChatsResponse(
                     threadId = thread.id,
                     chats = chats.map { ChatResponse(it.id, it.question,
     it.answer, it.createdAt) }
                 )
             }
         }

         // Thread 삭제 (soft delete)
         fun deleteThread(userId: Long, threadId: Long, isAdmin: Boolean) {
             val thread = threadRepository.findById(threadId)
                 .orElseThrow { NoSuchElementException("Thread not found: 
     $threadId") }
             if (!isAdmin && thread.userId != userId)
                 throw AccessDeniedException("권한 없음")
             thread.deletedAt = Instant.now()
             threadRepository.save(thread)
             threadCacheService.invalidate(thread.userId)
         }

         private fun buildSystemInstruction(chunks: List<String>): String {
             if (chunks.isEmpty()) return "당신은 도움이 되는 AI 
     어시스턴트입니다."
             val context = chunks.joinToString("\n\n") { "- $it" }
             return "당신은 증권 전문가 AI 어시스턴트입니다. 아래 참고 문서를 
     바탕으로 답변하세요:\n\n$context"
         }

         private fun buildContents(history: List<Chat>, question: String):
     List<Content> =
             buildList {
                 history.forEach { chat ->
                     add(content(role = "user") { text(chat.question) })
                     add(content(role = "model") { text(chat.answer) })
                 }
                 add(content(role = "user") { text(question) })
             }
     }

     10. ChatController

     @RestController
     class ChatController(private val chatService: ChatService, private val
     userRepository: UserRepository) {

         @PostMapping("/chats")
         fun createChat(
             @RequestBody req: ChatCreateRequest,
             authentication: Authentication
         ): SseEmitter {
             val userId = resolveUserId(authentication)
             val emitter = SseEmitter(0L)
             chatService.createChat(userId, req, emitter)
             return emitter
         }

         @GetMapping("/chats")
         @PreAuthorize("isAuthenticated()")
         fun listChats(
             authentication: Authentication,
             @RequestParam(defaultValue = "0") page: Int,
             @RequestParam(defaultValue = "10") size: Int,
             @RequestParam(defaultValue = "desc") sort: String
         ): Page<ThreadWithChatsResponse> {
             val user = resolveUser(authentication)
             val direction = if (sort == "asc") Sort.Direction.ASC else
     Sort.Direction.DESC
             val pageable = PageRequest.of(page, size, Sort.by(direction,
     "createdAt"))
             return chatService.listChats(user.id, user.role == Role.ADMIN,
     pageable)
         }

         @DeleteMapping("/threads/{threadId}")
         @ResponseStatus(HttpStatus.NO_CONTENT)
         fun deleteThread(@PathVariable threadId: Long, authentication: 
     Authentication) {
             val user = resolveUser(authentication)
             chatService.deleteThread(user.id, threadId, user.role ==
     Role.ADMIN)
         }

         private fun resolveUser(auth: Authentication): User =
             userRepository.findByEmail(auth.name) ?: throw
     UsernameNotFoundException(auth.name)

         private fun resolveUserId(auth: Authentication): Long =
     resolveUser(auth).id
     }
